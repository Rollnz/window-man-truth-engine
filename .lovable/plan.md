# WindowMan Truth Engine — Cursor Migration Kit Plan

## Goal
Produce a complete, non-destructive migration blueprint that lets you clone this Lovable project into a fresh Cursor-managed repo, wire up Supabase, and verify the backend without exposing secrets or touching production.

## Deliverables (created on approval)

I will create exactly these files in the repo — no destructive edits, no DB migrations, no secret values:

1. **`MIGRATION_BLUEPRINT.md`** — the master developer guide following your 11-section structure (Assumptions → Manifest → .env.example → Dashboard Mirror Checklist → Bundling Script → Env Validation → Smoke Test → Source-to-Target Checklist → Verification Protocol → Rollback Plan → Cursor Execution Prompt).
2. **`migration-manifest.json`** — generated by walking the actual file tree (not invented). Categorized by `root_config`, `src`, `src_integrations`, `src_components`, `src_pages`, `src_hooks`, `src_lib`, `public`, `supabase_config`, `supabase_functions`, `supabase_migrations`, `supabase_shared`, `scripts`, `tests`, `docs`, `unknown_or_review_required`. Each entry: `path`, `required`, `category`, `migration_notes`, `contains_secret_risk`, `browser_exposed`.
3. **`.env.example`** — frontend `VITE_*` block + clearly-separated "Edge Function Secrets (server-only)" block for Resend, Twilio, Meta CAPI, Nextdoor, internal app secrets. No real values.
4. **`scripts/bundle-migration.mjs`** — Node-only (uses `node:zlib` + `node:fs` tar.gz; no external deps). Cross-platform. Excludes `node_modules`, `dist`, `.git`, `.env*` (keeps `.env.example`), logs, caches, `migration-output/`. Prints include/exclude summary. Fails safe.
5. **`scripts/validate-env.mjs`** — Loads `.env.local`, validates required `VITE_*`, warns on optional integrations, scans `src/` for forbidden references to `SUPABASE_SERVICE_ROLE_KEY`, `TWILIO_AUTH_TOKEN`, `META_ACCESS_TOKEN`, `RESEND_API_KEY`. Pass/fail report.
6. **`scripts/smoke-test-edge-functions.mjs`** — Dry-run by default, `--live` flag required for real calls. Tests the actual edge functions present in this project (send-otp, verify-otp, verify-lead-otp, initiate-lead-verification, auth-email-hook, orchestrate-quote-analysis, wm-analyze-quote, upload-quote, score-event, signal, track, enqueue-phonecall, trigger-phone-call, slide-over-chat, expert-chat, etc.). Synthetic payloads only. Prints curl equivalents.

## Inspection I will do before writing the manifest
- Walk `src/`, `supabase/functions/`, `supabase/migrations/`, `public/`, `scripts/`, `e2e/`, root configs via `code--exec` with `find`/`ls`.
- Grep for `import.meta.env`, `Deno.env.get`, `supabase.functions.invoke`, and third-party SDK names to set the `contains_secret_risk` / `browser_exposed` flags accurately.
- Read `supabase/config.toml` to enumerate functions and `verify_jwt` overrides (flag any `verify_jwt = false` for manual review in the blueprint).

## Safety rails enforced in the output
- No real secrets emitted anywhere — placeholders only.
- No `supabase db push`, no migration execution, no destructive shell commands run.
- Server-only secret names never prefixed `VITE_`.
- Blueprint explicitly tells Cursor to **stop and request manual confirmation** before `supabase db push`, SQL restore, or production deploy.
- Edge functions with `verify_jwt = false` flagged for manual review as public-callable.
- `.env`, `.env.local`, `.env.*.local` excluded from the bundle.

## Cursor Execution Prompt (final section of blueprint)
A paste-ready prompt that instructs Cursor to: inspect the migrated repo, run install/build/lint, run `validate-env.mjs`, run `smoke-test-edge-functions.mjs` (dry-run only), confirm Supabase client env names match code, confirm function names match `supabase/config.toml`, emit a migration status report, and **halt** before any destructive Supabase command.

## Out of scope (will not do)
- Running the bundling/validation/smoke-test scripts.
- Pushing migrations or deploying functions.
- Editing existing source files.
- Modifying `.env`, `supabase/config.toml`, or `src/integrations/supabase/client.ts`.
